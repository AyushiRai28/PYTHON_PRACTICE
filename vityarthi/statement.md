# Problem Statement

Educational institutions often require a simple system to collect basic student and faculty information and provide academic details such as subjects and examination schedules. Manual tracking becomes inefficient, especially when dealing with multiple streams and different exam timelines.
This project aims to develop a Python-based interactive system that automates information collection and provides academic details quickly and accurately.


---

# Scope of the Project -->

The scope of this project includes:

Collecting personal details from users (students or faculty)

Validating inputs such as age and phone number

Displaying subject lists based on student streams (PCM, PCB, Arts, Commerce)

Displaying faculty details and assigned subjects

Providing an examination schedule for all streams

Showing exam eligibility criteria

Ensuring smooth user interaction and clear output structure


This project focuses on a console-based Python application without database storage or GUI.


---
# Target Users -->

* Students of Dumplin University

* Faculty members

* Administrative staff who need quick access to academic information

* New/Prospective students who want to understand subjects and exam schedules



---

# High-Level Features

1. User Type Detection

Accepts input for user type: student or faculty

Routes to the correct module



2. Student Information Module

-Takes personal details
-Validates age and phone number
-Displays stream-specific subjects



3. Faculty Information Module
-Accepts and displays faculty details such as subject and contact information



4. Examination Information Module
-Shows exam dates for all available streams
-Displays exam eligibility criteria



5. Structured Output
-Clean formatting
-Easy-to-read section-wise results
